# Writing Style Guide

A writing style guide establishes consistent standards for creating technical documentation. Consistency improves readability, reduces confusion, and provides a professional experience for users.

## Write for Your Audience

Always write with the reader in mind.

Consider:

- Technical knowledge
- Experience level
- Goals
- Tasks they need to accomplish

Avoid making assumptions about what users already know.

## Use Simple Language

Prefer clear, everyday words over complex terminology whenever possible.

Instead of:

> Utilize the configuration utility.

Write:

> Use the configuration tool.

Simple language improves comprehension for both native and non-native English speakers.

## Use Active Voice

Active voice makes instructions easier to understand.

Example:

❌ The report should be generated by the system.

✅ The system generates the report.

## Keep Sentences Short

Long sentences are difficult to read.

Aim to:

- Express one idea per sentence.
- Keep paragraphs focused on a single topic.
- Break complex procedures into numbered steps.

## Use Consistent Terminology

Choose one term for each concept and use it consistently.

For example:

- Always use **Sign in** instead of alternating between **Log in**, **Login**, and **Authenticate**.
- Use the official product names and feature names.

## Format Lists Clearly

Use:

- Bullet lists for related information.
- Numbered lists for procedures.
- Tables when comparing information.

Consistent formatting helps users scan documentation quickly.

## Highlight Important Information

Use formatting appropriately.

- **Bold** for important terms.
- `Code formatting` for commands, filenames, and configuration values.
- Notes and warnings only when necessary.

Avoid excessive formatting, as it reduces its effectiveness.

## Summary

A consistent writing style improves the quality of technical documentation by making it easier to read, understand, and maintain. A well-defined style guide also helps multiple authors produce documentation with a consistent voice and structure.
